About Chrono Timer

Overview

Timing for many sports events involves recording a start time and end time to yield a time for a run, leg or lap. This can be used in ski races, swim meets, bobsled runs, or similar events. In all cases, a start is generated by either a start gate (or electric eye) or a manual start, and the finish is recorded when the competitor trips the sensor at the end. The sensor could be an electric eye or a touch pad. A finish signal can also be generated manually. Commands can be read from the console, a file, or by manual operation of device


Units of time

Timing is done in hundredths of seconds ranging from 0.00 to 9999.99 seconds.


Releases

Release 1.0

The first release of the timing system allows one stream of racers that can start, finish, cancel and not finish (DNF). These are individual times (not parallel or group).

No GUI required.

Release 2.0

New features for second release include multiple channels and export of data to the file (connected on USB) and display on console.

Basic GUI

Release 3.0

Additional features and maintenance including various displays.

Ability to handle single start, multiple finishes.

Advanced GUI.

Release 4.0

Additional features handling analytics


Comands

ON
     Turn system on, enter quiescent state

OFF
     Turn system off (but stay in simulator)

EXIT
     Exit the simulator

RESET
     Resets the System to initial state

TIME [hour]:[min]:[sec]
     Set the current time

TOG [channel]
     Toggle the state of the channel [CHANNEL]

CONN [sensor]
     Connect a type of sensor to channel [NUM] [sensor] = {EYE, GATE, PAD}

DISC [NUM]
     Disconnect a sensor from channel

EVENT [TYPE]
     IND | PARIND | GRP | PARGRP

NEWRUN
     Create a new Run (must end a run first)

ENDRUN
      Done with a Run

PRINT
      [RUN] Print the run on stdout

EXPORT
      [RUN] Export run in XML to file “RUN”

NUM [NUMBER]
      Set [NUMBER] as the next competitor to start.

CLR [NUMBER]
      Clear [NUMBER] as the next competitor

SWAP
      Exchange next two competitors to finish

RCL
      Recall last trigger event

START
      Start trigger channel 1

FINISH
      Finish trigger channel 2

TRIG ]NUM]
     Trigger channel [NUM]
